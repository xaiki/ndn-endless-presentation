<2016-10-11 Tue> Meeting Questions

Performance

Timing face.processEvents for each packet makes it that we get a 200-300ms lag between interest and data production for each 4k packet, it's extremely slow. I could make a scheduler to accelerate polling when transfering but there must be a better solution

Chunk/Packet Size

On the first phase of the deployement we're going to keep our S3/HTTP infrastructure basically pristine. We need, at the edge of the NDN network, to transform that data, transform the files hosted on HTTP in a steram of NDN packets. As we care a lot about not downloading things twice i need a mechanism to make sure that we get the right bytes, i could trust HTTP range, but we all know that's not good enough. How'd you implement chunking at a 15Mb size-range (i think it was prooven a 'good choice' for torrents) ?

Crypto and Signing

Corollary to the previous question, is there any established mechanism to pre-sign chunks of data and fetch them from HTTP
